Enterprise RAG fails when retrieval returns the wrong category — category bleed creates hallucination paths agents cannot detect. PrismRAG applies deterministic category projection: your rules define the graph, every chunk is auditable, Graph RAG runs on your taxonomy. # Your mapping — not document statistics
from prismrag_patch import PrismRAG

rag = PrismRAG(
  mapping={
    "categories": [
      { "slug": "risk",   "label": "Risk & Compliance" },
      { "slug": "growth", "label": "Growth" }

      { "word": "volatility", "category_slug": "risk" }
    ]
  },

results = rag.search("compliance exposure", top_k=5)

FAQ

Enterprise RAG questions architects ask

Direct answers for Graph RAG vs taxonomy enforcement, category bleed, and audit-ready retrieval.

How to stop RAG category bleed?

Define your taxonomy (categories + keyword rules), then remap every chunk and query embedding with deterministic category projection before indexing. PrismRAG Tier-1 separates categories in cosine space while preserving semantic content — with a full audit trail from rule → chunk → retrieval.

Graph RAG vs client-defined taxonomy — what's the difference?

Graph RAG learns edges from document co-occurrence. Client-defined taxonomy puts your expert categories first — same corpus, different graphs per tenant. PrismRAG inverts the pipeline: mapping → embed → graph, not documents → statistics → graph.

What is deterministic category projection?

A closed-form vector blend: infer category from weighted rules, nudge the embedding toward a category-specific direction on the unit sphere (α=0.35 default). No training, no randomness, no extra API at projection time. Read the whitepaper →

How does PrismRAG prevent AI agent hallucination?

Agents hallucinate when retrieval returns wrong-category chunks. PrismRAG anchors every chunk to a mapping rule you wrote — MCP-native search returns verified, category-placed context.

Can I use PrismRAG with my existing vector database?

Yes. prismrag-patch on PyPI wraps pgvector, ChromaDB, Pinecone, and Weaviate. Any embedding model — you provide vectors, the library remaps and stores them in your DB.
